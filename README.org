* Introduction

This guide explains how to build a small HPC cluster from scratch using
VirtualBox and Rocky Linux.

The setup is designed for learning and experimentation. The overall structure
is inspired by real HPC environments and supercomputers, but everything is
kept lightweight so it can run on a normal laptop or workstation.

The goal is to create a working cluster environment where you can practice:

- Linux system administration
- Slurm job scheduling
- MPI/OpenMP execution
- Cluster networking
- Shared storage setup
- User management
- Software installation
- HPC monitoring and automation

This is not a theoretical HPC book. The focus is completely practical.

Throughout this guide, we will gradually build the cluster node-by-node and
configure each component manually.

* Cluster Architecture
The cluster consists of five virtual machines connected using an internal
network.

#+begin_example
                 HOST MACHINE
                       |
         ssh -p XXXX localhost

compute01   10.10.10.11
compute02   10.10.10.12
compute03   10.10.10.13
#+end_example

* Node Details
** Node Role
*** Master Node

The master node manages the overall cluster.

Main responsibilities:

- Slurm controller
- NFS server
- LDAP server
- Monitoring tools
- Shared software stack
- Cluster configuration

*** Login Node

The login node is the access point for users.

Typical tasks performed here:

- SSH login
- Code compilation
- Job submission
- Accessing shared storage

*** Compute Nodes

Compute nodes run the actual workloads.

These nodes are mainly used for:

- MPI jobs
- OpenMP jobs
- Batch workloads
- Parallel execution

** Networking Setup

Two network adapters are used for each VM.

*** NAT Adapter

Used for:

- Internet access
- Package installation
- SSH access from host system

*** Internal Network

Used for communication between cluster nodes.

Network name:

#+begin_example
hpcnet
#+end_example

This network is isolated and only used inside the cluster.

** Static IP Addressing

Each node uses a fixed IP address.

| VM Name   | Hostname        | Internal IP | SSH Port |
|-----------+-----------------+-------------+----------|
| master    | master.local    | 10.10.10.1  | 2221     |
| login     | login.local     | 10.10.10.2  | 2222     |
| compute01 | compute01.local | 10.10.10.11 | 2223     |
| compute02 | compute02.local | 10.10.10.12 | 2224     |
| compute03 | compute03.local | 10.10.10.13 | 2225     |

Example SSH access:

#+begin_src bash
ssh -p 2221 hpcuser@localhost
#+end_src

** Install VirtualBox on Manjaro
I'm using Manjaro (Arch) as my base system.
*** Update packages
#+begin_src bash
sudo pacman -Syu
#+end_src

*** Install VirtualBox and host modules
#+begin_src bash
sudo pacman -S virtualbox virtualbox-host-modules-arch
#+end_src

*** Install VirtualBox extension pack
#+begin_src bash
sudo pacman -S virtualbox-ext-oracle
#+end_src

*** Load VirtualBox kernel module
#+begin_src bash
sudo modprobe vboxdrv
#+end_src

*** Add current user to VirtualBox group
#+begin_src bash
sudo usermod -aG vboxusers $USER
#+end_src

*** Reboot the system
#+begin_src bash
sudo reboot
#+end_src

** Initial System Verification
*** Check Interfaces

#+begin_src bash
ip addr
#+end_src

Expected:

- NAT interface should receive:
  - 10.0.2.15
- Internal interface should exist but may not yet have static IP

** Configure Static Internal Network
*** Check Device Names

#+begin_src bash
nmcli device status
#+end_src

Expected:

| Interface | Purpose |
|------------+----------|
| enp0s3     | NAT |
| enp0s8     | Internal Network |

** Configure Internal Interface

Launch NetworkManager TUI:

#+begin_src bash
sudo nmtui
#+end_src

Navigate:

#+begin_example
Edit a connection
#+end_example

Select:

#+begin_example
enp0s8
#+end_example

Set:

| Field | Value |
|--------+-------|
| IPv4 Method | Manual |
| Address | 10.10.10.1/24 |
| Gateway | blank |
| DNS | blank |

DO NOT modify NAT interface.

** Restart Networking

#+begin_src bash
sudo systemctl restart NetworkManager
#+end_src

** Verify Networking

#+begin_src bash
ip addr
#+end_src

Expected:

| Interface | IP |
|------------+----------------|
| NAT        | 10.0.2.x |
| Internal   | 10.10.10.1 |

** Configure /etc/hosts

Edit:

#+begin_src bash
sudo vim /etc/hosts
#+end_src

Add:

#+begin_example
10.10.10.1   master master.local
10.10.10.2   login login.local
10.10.10.11  compute01 compute01.local
10.10.10.12  compute02 compute02.local
10.10.10.13  compute03 compute03.local
#+end_example
This allows hostname-based communication between nodes.

** Disable Firewall (Temporary)
For initial setup and testing:
#+begin_src bash
sudo systemctl disable --now firewalld
#+end_src
NOTE:

Firewall rules can be configured properly later after cluster setup is complete.

** SELinux Configuration

Check status:

#+begin_src bash
getenforce
#+end_src

Temporarily disable enforcing:

#+begin_src bash
sudo setenforce 0
#+end_src

Persist configuration:

#+begin_src bash
sudo vim /etc/selinux/config
#+end_src

Modify:

#+begin_example
SELINUX=permissive
#+end_example
This helps avoid permission-related issues during initial cluster setup.

** Verify Hostname
#+begin_src bash
hostnamectl
#+end_src

Expected:

#+begin_example
Static hostname: master.local
#+end_example

If hostname is not updated(showing previous), reboot the machine.

At this point, the master node setup is complete and ready for cloning or
further cluster configuration.
